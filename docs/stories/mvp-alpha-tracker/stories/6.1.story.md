## Description

Build the complete Dashboard view by composing `useDashboard`, `PnLSummaryCard`, `PeriodToggle`, `PnLBarChart`, and replacing the Wave 6 stub in `DashboardPage`.

- **`useDashboard.ts`** — TanStack Query hook that queries Concluidas only, aggregates PnL by period using `dates.ts` helpers, and returns chart-ready data plus pending count.
- **`PnLSummaryCard.tsx`** — Displays total PnL for the selected period with green/red coloring.
- **`PeriodToggle.tsx`** — 3-button group (Diário/Semanal/Mensal) with local active state.
- **`PnLBarChart.tsx`** — Recharts BarChart rendering PnL over time with green/red bars. Recharts imported here only — not in the global bundle.
- **`DashboardPage.tsx`** — Replaces stub. Composes all components and owns period state.

Only Concluidas aggregate into totals. Pendentes are shown separately (count only) and never included in PnL total.

## Acceptance Criteria

- [x] **AC-1:** `src/hooks/useDashboard.ts` exports `useDashboard(period: DashboardPeriod)`:
  - **Type:** `type DashboardPeriod = 'day' | 'week' | 'month'`
  - **Query:** `useQuery({ queryKey: ['dashboard', period], queryFn: ... })` that:
    - Fetches from `operacoes` with `select('*, apostas(*, casa:casas(*))')` (same join pattern as useOperacoes)
    - Uses `const { data: result, error }` alias to avoid `operacoes.data` naming collision
    - Filters only `status === 'Concluida'` in-memory for period aggregation
    - Uses `isInPeriod(operacao.data, period)` from `@/lib/dates` to filter by selected period
  - **Returns:**
    ```typescript
    {
      totalPnl: number                        // sum of pnl for Concluidas in period
      chartData: { label: string; pnl: number }[]  // grouped by day for week/month; by operation for day
      pendentesCount: number                  // count of all Pendente operations (not period-filtered)
      isLoading: boolean
      isError: boolean
    }
    ```
  - **Chart data logic:**
    - `period === 'day'`: one entry per Concluida in today, `label = operacao.tipo`
    - `period === 'week'`: group Concluidas by day using `format(parseISO(op.data + 'T00:00:00'), 'EEE', { locale: ptBR })` → `{ label: 'Seg', pnl: sum }`
    - `period === 'month'`: group Concluidas by day → `{ label: format(parseISO(op.data + 'T00:00:00'), 'dd/MM'), pnl: sum }`
  - **Note on date-fns locale:** `import { ptBR } from 'date-fns/locale'` for Portuguese day abbreviations

- [x] **AC-2:** `src/components/dashboard/PnLSummaryCard.tsx` exports `PnLSummaryCard`:

## Technical Notes

- **`queryKey: ['dashboard', period]`:** Scoped by period so TanStack Query caches separately per period. Use `invalidateQueries({ queryKey: ['dashboard'] })` (already done in useOperacoes delete/marcarResultado) to bust all dashboard cache on mutations.
- **Date collision alias:** `const { data: result, error } = await supabase.from('operacoes')...` — always alias to avoid `operacoes.data` collision with Supabase response.
- **`operacao.data + 'T00:00:00'`:** Required in `parseISO` call for date-only strings — prevents UTC timezone shift. Example: `parseISO('2026-05-18T00:00:00')` gives correct local midnight (consistent with Wave 5 date pattern).
- **recharts import isolation:** Recharts is only imported in `PnLBarChart.tsx`. This file is only rendered inside `DashboardPage`. When `DashboardPage` is eventually lazy-loaded in router.tsx (MVP-6-4), recharts will be code-split automatically.
- **date-fns ptBR locale:** `import { ptBR } from 'date-fns/locale'` — available via existing `date-fns` dependency.
- **`format` from date-fns:** Import alongside existing utilities: `import { format, parseISO } from 'date-fns'`
- **PnL aggregation (sum):** Use `reduce` with `+` for simple aggregation (no Decimal.js needed — display only, not stored; floating-point cosmetic drift acceptable for dashboard totals).
- **Recharts `Cell` pattern:**
  ```tsx
  <Bar dataKey="pnl">
    {data.map((entry, index) => (
      <Cell key={index} fill={entry.pnl > 0 ? '#16a34a' : '#dc2626'} />
    ))}
  </Bar>
  ```
- **Spec refs:** FR-9 (dashboard PnL diário/semanal/mensal + filtro por tipo), NFR-2 (recharts isolated for bundle)
