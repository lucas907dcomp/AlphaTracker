## Description

Cria a infraestrutura de banco de dados necessária para o sistema de parceiros e split de lucro. Esta story é pré-requisito bloqueante para todas as outras stories da Wave 7.

Quatro migrations SQL devem ser aplicadas no Supabase (via SQL Editor, pois `db push` está bloqueado pela rede da empresa):

1. `006_create_parceiros` — Tabela de parceiros com percentual de split
2. `007_add_parceiro_to_casas` — FK opcional em casas vinculando a um parceiro
3. `008_add_origem_operacoes` — Dois novos campos em operações: origem da freebet e custo de liberação
4. `009_create_parceiro_repasses` — Tabela de registro de pagamentos ao parceiro

---

## Acceptance Criteria

- [ ] **AC-1:** Migration `006_create_parceiros.sql` aplicada com sucesso:
  ```sql
  CREATE TABLE parceiros (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    nome TEXT NOT NULL,
    percentual NUMERIC(5,2) NOT NULL CHECK (percentual > 0 AND percentual < 100),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
  ALTER TABLE parceiros ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "users see own parceiros" ON parceiros
    FOR ALL USING (auth.uid() = user_id);
  ```

- [ ] **AC-2:** Migration `007_add_parceiro_to_casas.sql` aplicada:
  ```sql
  ALTER TABLE casas ADD COLUMN parceiro_id UUID REFERENCES parceiros(id) ON DELETE SET NULL;
  ```

- [ ] **AC-3:** Migration `008_add_origem_operacoes.sql` aplicada:
  ```sql
  ALTER TABLE operacoes ADD COLUMN operacao_origem_id UUID REFERENCES operacoes(id) ON DELETE SET NULL;
  ALTER TABLE operacoes ADD COLUMN custo_liberacao NUMERIC(10,2);
  ```

- [ ] **AC-4:** Migration `009_create_parceiro_repasses.sql` aplicada:
  ```sql
  CREATE TABLE parceiro_repasses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
    parceiro_id UUID NOT NULL REFERENCES parceiros(id) ON DELETE CASCADE,
    valor NUMERIC(10,2) NOT NULL CHECK (valor > 0),
    data DATE NOT NULL,
    notas TEXT,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
  );
  ALTER TABLE parceiro_repasses ENABLE ROW LEVEL SECURITY;
  CREATE POLICY "users see own repasses" ON parceiro_repasses
    FOR ALL USING (auth.uid() = user_id);
  ```

- [ ] **AC-5:** Todos os arquivos de migration existem em `supabase/migrations/` (para histórico local)

- [ ] **AC-6:** `npm run typecheck` passa sem erros após as migrations (sem código novo ainda, só arquivos SQL)

## Technical Notes

- **CRÍTICO:** `db push` está bloqueado pela rede da empresa. As migrations devem ser aplicadas manualmente no SQL Editor do Supabase: https://supabase.com/dashboard/project/clmkcxsrxumnernbdepd/sql/new
- Os arquivos SQL devem existir localmente em `supabase/migrations/` para manter histórico, mas a aplicação é manual
- `parceiros.percentual` usa `CHECK (percentual > 0 AND percentual < 100)` — o dono recebe o restante implicitamente
- `operacoes.operacao_origem_id` é auto-referencial (FK para a própria tabela `operacoes`)
- `casas.parceiro_id` usa `ON DELETE SET NULL` para não deletar casas se o parceiro for removido
