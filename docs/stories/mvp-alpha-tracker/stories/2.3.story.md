## Technical Notes

- **RLS:** The `casas` table has `USING (auth.uid() = user_id) WITH CHECK (auth.uid() = user_id)` â€” the Supabase client automatically filters results to the authenticated user. No manual `where` clause needed for the user filter, but `is_active = true` must be explicit.
- **Soft delete pattern:** `is_active = false` preserves historical references from `apostas.casa_id`. Hard delete is intentionally blocked by `ON DELETE RESTRICT` (set by @data-engineer in MVP-1-4).
- **Query key convention:** Use `['casas']` as the query key throughout â€” Wave 4 (`useCasas` in `LegRow`) expects this key for dropdown invalidation.
- **Supabase query:** `supabase.from('casas').select('*').eq('is_active', true).order('nome')` â€” returns `{ data, error }`. Check `error` and throw if present.
- **casaSchema import:** Import from `@/schemas/casaSchema` (created in MVP-1-3). The schema validates `nome.min(1).trim()`.
- **TanStack Query v5 mutations:** Use `mutateAsync` or `mutate`; handle errors in `onError` callback or via try/catch in the form `handleSubmit`.
- **Spec ref:** FR-1, Section 3.3 (CasaForm, CasaList components), Section 3.4 (casas table schema).

```typescript
// useCasas.ts pattern
export function useCasas() {
  const queryClient = useQueryClient()

  const { data: casas = [], isLoading, isError } = useQuery({
    queryKey: ['casas'],
    queryFn: async () => {
      const { data, error } = await supabase
        .from('casas')
        .select('*')
        .eq('is_active', true)
        .order('nome')
      if (error) throw error
      return data as Casa[]
    },
  })

  const { mutateAsync: createCasa } = useMutation({
    mutationFn: async (nome: string) => {
      const { error } = await supabase.from('casas').insert({ nome })
      if (error) throw error
    },
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['casas'] }),
  })

  const { mutateAsync: deactivateCasa } = useMutation({
    mutationFn: async (id: string) => {
      const { error } = await supabase.from('casas').update({ is_active: false }).eq('id', id)
      if (error) throw error
    },
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['casas'] }),
  })

  return { casas, isLoading, isError, createCasa, deactivateCasa }
}
```
