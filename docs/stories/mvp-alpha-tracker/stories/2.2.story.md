## Description

Implement the `LoginPage` with email/password authentication form, and create the three base UI components (`Input`, `Button`, `Select`) used throughout the application. The `Input` component must support `onKeyDown` for Tab/Enter keyboard navigation (NFR-1). `LoginPage` shows a `sonner` toast on auth errors and redirects to `/dashboard` on success.

## Technical Notes

- **sonner:** `import { Toaster } from 'sonner'` in the app root, `import { toast } from 'sonner'` where used. Toaster must be in the React tree above all routes.
- **NFR-1 compliance:** `Input` component's `onKeyDown` is the foundation for Tab/Enter flow in `LegRow` (Wave 4). The component must forward the prop correctly — do not swallow it.
- **Tailwind forms plugin:** `@tailwindcss/forms` is already installed and configured in `tailwind.config.ts` — it provides base form element resets that make Tailwind classes apply cleanly on `<input>` and `<select>`.
- **Spec ref:** Component architecture — `spec.md` Section 3.3, Auth flow — Section 3.5
- **LoginPage redirect logic:** If `session` is already set when `LoginPage` mounts, redirect immediately to `/dashboard` (user already logged in).

```typescript
// LoginPage pattern
export default function LoginPage() {
  const { signIn, session } = useAuth()
  const navigate = useNavigate()
  const [loading, setLoading] = useState(false)

  useEffect(() => {
    if (session) navigate('/dashboard', { replace: true })
  }, [session, navigate])

  async function handleSubmit(e: React.FormEvent<HTMLFormElement>) {
    e.preventDefault()
    setLoading(true)
    const { error } = await signIn(email, password)
    if (error) toast.error(error.message)
    else navigate('/dashboard', { replace: true })
    setLoading(false)
  }
  // ...
}
```
