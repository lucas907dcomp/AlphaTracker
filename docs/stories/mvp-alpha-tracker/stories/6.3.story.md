## Description

Implement the 5 mandatory test scenarios that verify end-to-end correctness of the system. These tests cover the 4 highest-risk integration paths:

1. **Tab/Enter keyboard flow** — Playwright E2E test verifying NFR-1 (3-leg operation registered without mouse).
2. **RLS data isolation** — Supabase test client verifying user B cannot see user A data (NFR-3, CON-4).
3. **Dashboard cache invalidation** — TanStack Query cache updates dashboard total after new operation, without page refresh.
4. **Asymmetric PnL** — Vitest integration test verifying `calcularCenariosPnL` with 3 legs of different grossReturn.
5. **Draft recovery** — Zustand persist: draft survives localStorage clear/reload cycle.

Tests 1, 2, 3 require Supabase Cloud for full execution (REL-001). Tests 4 and 5 run locally with `npx vitest run`. Playwright test 1 requires a running dev server.

## Technical Notes

- **REL-001 applies to tests 1, 2, 3:** These require Supabase Cloud. Use `.skip` patterns consistent with Wave 4 CONCERNS gate. Tests 4 and 5 are self-contained and run offline.
- **Test directory structure:** Create `src/tests/e2e/` for Playwright specs and `src/tests/integration/` for Vitest integration tests. This separates E2E (browser-based) from integration (Node/jsdom).
- **vitest config for integration tests:** Existing vitest config (from MVP-3-2) should pick up `src/tests/integration/**`. Verify `include` pattern in `vite.config.ts` or create `vitest.config.ts` if needed.
- **Asymmetric PnL test data (AC-5):** Apostas must match the `Aposta` type interface (snake_case fields):
  ```typescript
  const apostas: Aposta[] = [
    { id: '1', operacao_id: 'op1', casa_id: 'c1', stake: 100, gross_return: 200, is_freebet: false, is_double_green: false, resultado: 'Pendente', created_at: '', updated_at: '' },
    { id: '2', operacao_id: 'op1', casa_id: 'c2', stake: 20, gross_return: 85, is_freebet: false, is_double_green: false, resultado: 'Pendente', created_at: '', updated_at: '' },
    { id: '3', operacao_id: 'op1', casa_id: 'c3', stake: 39, gross_return: 85, is_freebet: false, is_double_green: false, resultado: 'Pendente', created_at: '', updated_at: '' },
  ]
  ```
- **Zustand persist test (AC-6):** Zustand + vitest/jsdom: localStorage is available in jsdom. The `persist` middleware reads from `localStorage` on store initialization. To test recovery: call `setDraft`, verify `localStorage.getItem('operacao-draft')` contains the serialized state, then reinitialize the store from scratch.
- **Playwright deps:** `playwright` is in `package.json` dependencies from Wave 1. Run `npx playwright install chromium` if browser binaries are not yet installed.
- **Spec refs:** NFR-1 (Tab/Enter test), NFR-3 (RLS isolation), NFR-5 (draft recovery), FR-5 (PnL assimetrico)
