## Description

Implement `useOperacoes` — the central TanStack Query hook for all operations CRUD. Covers: listing operations with their apostas (FK join including casa names), creating a new operation + its apostas in two sequential inserts, deleting an operation (apostas cascade-deleted via FK), and `marcarResultado` — marks the winner leg, calculates definitive pnl via `calcularCenariosPnL`, and transitions status `Pendente → Concluida`. Both mutating operations invalidate `['operacoes']` and `['dashboard']` query keys on success.

- [ ] **AC-3:** `deleteOperacao(id: string): Promise<void>` mutation:
  - `supabase.from('operacoes').delete().eq('id', id)`
  - Apostas are cascade-deleted by `ON DELETE CASCADE` FK — no manual aposta deletion needed
  - On success: `invalidateQueries({ queryKey: ['operacoes'] })` + `invalidateQueries({ queryKey: ['dashboard'] })`

- [ ] **AC-4:** `marcarResultado({ operacaoId, winnerApostaId, apostas }: MarcarResultadoParams): Promise<void>` mutation:
  - Type: `type MarcarResultadoParams = { operacaoId: string; winnerApostaId: string; apostas: Aposta[] }`
  - Step 1 — Update each aposta resultado individually (no batch UPDATE with different values in Supabase JS):
    - Winner aposta: `resultado: 'Ganhou'`
    - All other apostas: `resultado: 'Perdeu'`
  - Step 2 — Calculate final pnl:
    - `const cenarios = calcularCenariosPnL(apostas)` — uses the Aposta[] passed as param
    - `const winnerAposta = apostas.find(a => a.id === winnerApostaId)!`
    - `const finalPnl = cenarios.find(c => c.casaId === winnerAposta.casa_id)?.pnl ?? 0`
  - Step 3 — Update operacao: `{ pnl: finalPnl, status: 'Concluida' }`
  - On success: `invalidateQueries({ queryKey: ['operacoes'] })` + `invalidateQueries({ queryKey: ['dashboard'] })`

## Technical Notes

- **TanStack Query v5 pattern:** Same as `useCasas` — `useQuery` + `useMutation` from `@tanstack/react-query`. Refer to `src/hooks/useCasas.ts` as the existing pattern.
- **`operacoes.data` naming collision (CRITICAL):** The `data` column in the `operacoes` table conflicts with the `{ data, error }` destructuring pattern from Supabase JS. Always destructure as `const { data: result, error } = await supabase.from('operacoes')...` then use `result`.
- **camelCase → snake_case mapping (CRITICAL):** `OperacaoFormData` (from operacaoSchema) uses camelCase: `casaId`, `grossReturn`, `isFreebet`, `isDoubleGreen`. DB columns are snake_case. Map explicitly: `casa_id: leg.casaId`, `gross_return: leg.grossReturn`, `is_freebet: leg.isFreebet`, `is_double_green: leg.isDoubleGreen`.
- **Sequential inserts (no DB transaction):** Supabase JS browser client does not support `BEGIN/COMMIT`. Use sequential inserts: operacao first, then apostas. If apostas insert fails after operacao insert, the orphaned operacao will be left in the DB. Acceptable for MVP; consider cleanup in Wave 6.
- **`marcarResultado` — apostas passed as param:** Accept the already-loaded `Aposta[]` from the query cache rather than re-fetching from DB. Caller (`ResultadoModal`, MVP-5-1) passes the apostas from the loaded operacao.
- **`marcarResultado` — individual UPDATE loop:** Supabase JS does not support `UPDATE ... WHERE id IN (...) SET resultado = CASE WHEN id = $X THEN 'Ganhou' ...`. Use a loop or two separate updates (winner + losers).
- **`invalidateQueries` for dashboard:** Wave 6 `useDashboard` uses `queryKey: ['dashboard']`. Invalidating it here ensures the dashboard reflects the new pnl immediately after marking a result, without requiring a page reload.
- **RLS:** All queries are scoped by RLS — no manual `user_id` filter needed in the query or inserts. Apostas policy uses `EXISTS (SELECT 1 FROM operacoes WHERE id = apostas.operacao_id AND user_id = auth.uid())`.
- **Import paths:** `@/lib/supabase`, `@/lib/pnl`, `@/types`, `@/schemas/operacaoSchema`

```typescript
// useOperacoes.ts — reference structure
import { useQuery, useMutation, useQueryClient } from '@tanstack/react-query'
import { supabase } from '@/lib/supabase'
import { calcularCenariosPnL } from '@/lib/pnl'
import type { Operacao, Aposta } from '@/types'
import type { OperacaoFormData } from '@/schemas/operacaoSchema'

type MarcarResultadoParams = {
  operacaoId: string
  winnerApostaId: string
  apostas: Aposta[]
}

export function useOperacoes() {
  const queryClient = useQueryClient()

  const { data: operacoes = [], isLoading, isError } = useQuery({
    queryKey: ['operacoes'],
    queryFn: async () => {
      const { data: result, error } = await supabase
        .from('operacoes')
        .select('*, apostas(*, casa:casas(*))')
        .order('data', { ascending: false })
      if (error) throw error
      return result as Operacao[]
    },
  })

  const { mutateAsync: createOperacao } = useMutation({
    mutationFn: async (data: OperacaoFormData) => {
      const { data: opData, error: opError } = await supabase
        .from('operacoes')
        .insert({
          tipo: data.tipo,
          data: data.data,
          valor_pago_fixo: data.valorPagoFixo ?? null,
          notas: data.notas ?? null,
          status: 'Pendente',
        })
        .select('id')
        .single()
      if (opError) throw opError

      const apostasToInsert = data.legs.map(leg => ({
        operacao_id: opData.id,
        casa_id: leg.casaId,
        stake: leg.stake,
        gross_return: leg.grossReturn,
        is_freebet: leg.isFreebet,
        is_double_green: leg.isDoubleGreen,
      }))

      const { error: apostasError } = await supabase.from('apostas').insert(apostasToInsert)
      if (apostasError) throw apostasError
    },
    onSuccess: () => queryClient.invalidateQueries({ queryKey: ['operacoes'] }),
  })

  // deleteOperacao, marcarResultado follow the same pattern...

  return { operacoes, isLoading, isError, createOperacao, deleteOperacao, marcarResultado }
}
```
