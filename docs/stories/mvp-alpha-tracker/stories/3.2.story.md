  - Assertion: `expect(result[0].pnl).toBe(80)` and `expect(result[1].pnl).toBe(-35)`

- [ ] **AC-3:** **Test 2 — Freebet SNR** (Fórmula 3):
  - 2 apostas: `{ stake: 100, gross_return: 200, is_freebet: false }` and `{ stake: 20, gross_return: 85, is_freebet: true }`
  - `totalCusto = 100` (freebet excluded)
  - Expected: `[{ pnl: 100 }, { pnl: -15 }]`
  - Assertion: `expect(result[0].pnl).toBe(100)` and `expect(result[1].pnl).toBe(-15)`

- [ ] **AC-4:** **Test 3 — Assimétrico 3 cenários** (Fórmula 2):
  - 3 apostas (values from spec.md FR-7 example):
    - `{ stake: 100, gross_return: 200, is_freebet: false, is_double_green: false }` (Casa1)
    - `{ stake: 20, gross_return: 85, is_freebet: false, is_double_green: false }` (Casa2)
    - `{ stake: 39, gross_return: 85, is_freebet: false, is_double_green: false }` (Casa3)
  - `totalCusto = 159`
  - Expected: `[{ pnl: 41 }, { pnl: -74 }, { pnl: -74 }]`
  - Assertion: verify all 3 `pnl` values

- [ ] **AC-5:** **Test 4 — Duplo Green** (Fórmula 4):
  - 2 apostas: `{ stake: 100, gross_return: 200, is_double_green: false }` and `{ stake: 20, gross_return: 85, is_double_green: true }`
  - `totalCusto = 120`
  - Leg1 (no DG): `gr_efetivo = 200`, `pnl = 200 - 120 = 80`
  - Leg2 (DG): `gr_efetivo = 85 * 2 = 170`, `pnl = 170 - 120 = 50`
  - Expected: `[{ pnl: 80 }, { pnl: 50 }]`

- [ ] **AC-6:** **Test 5 — Precisão decimal** (CON-3 verification):
  - 2 apostas: `{ stake: 0.1, gross_return: 0.2 }` and `{ stake: 0.2, gross_return: 0.3 }`
  - `totalCusto = 0.1 + 0.2 = 0.3` (would be `0.30000000000000004` with native JS float)
  - Leg1: `pnl = 0.2 - 0.3 = -0.1` (exactly, not `-0.09999999999999998`)
  - Leg2: `pnl = 0.3 - 0.3 = 0` (exactly)
  - Assertion: `expect(result[0].pnl).toBe(-0.1)` and `expect(result[1].pnl).toBe(0)`
  - This test FAILS if native float arithmetic is used — it proves decimal.js is correctly applied

- [ ] **AC-7:** **Test 6 — N legs** (4 casas + 5 casas):
  - Sub-test A (4 legs): `stake=50, gr=200` × 4 apostas → `totalCusto=200`, all `pnl=0`
  - Sub-test B (5 legs): `stake=20, gr=100` × 5 apostas → `totalCusto=100`, all `pnl=0`
  - Assertions: `result.every(c => c.pnl === 0)` for both sub-tests

- [ ] **AC-8:** All 6 tests **PASS** when running `npx vitest run src/lib/__tests__/pnl.test.ts`. Zero failing tests, zero skipped tests.

## Technical Notes

- **Vitest path alias:** The `vite.config.ts` sets `@/` alias; Vitest inherits this config automatically (no separate `vitest.config.ts` needed). Import as `import { calcularCenariosPnL } from '@/lib/pnl'`.
- **Test helper:** Create a `makeAposta` helper inside the test file to reduce boilerplate:
  ```typescript
  function makeAposta(overrides: { stake: number; gross_return: number } & Partial<Aposta>): Aposta {
    return {
      id: crypto.randomUUID(),
      operacao_id: 'op-id',
      casa_id: 'casa-1',
      is_freebet: false,
      is_double_green: false,
      resultado: 'Pendente',
      created_at: '2026-01-01T00:00:00Z',
      updated_at: '2026-01-01T00:00:00Z',
      ...overrides,
    }
  ```

- **`casaId` and `casaNome` in assertions:** Tests should verify `pnl` values; `casaId` and `casaNome` are structural outputs — verify them in Test 1 at minimum to confirm mapping correctness, but they are not the focus of financial correctness.
- **HARD GATE logic:** The @qa gate for this story requires ZERO test failures. If any test fails, the story receives QA FAIL and returns to @dev. Wave 4 does not start until this gate is PASS.
- **`npm run test:run` vs `npx vitest run`:** Both work; the story verification uses `npx vitest run src/lib/__tests__/pnl.test.ts` to run only the P0 tests file.
