## Description

Implement the authentication hook (`useAuth`) and React Router setup with protected routes. This story wires together Supabase Auth session management with the SPA routing layer — unauthenticated users are always redirected to `/login`, authenticated users land on `/dashboard`. The `App.tsx` and `main.tsx` entry points are also configured here with `QueryClientProvider` and `RouterProvider`.

## Technical Notes

- **Auth pattern:** `supabase.auth.onAuthStateChange` returns an unsubscribe function — call it in the `useEffect` cleanup to avoid memory leaks.
- **Session persistence:** Supabase JS v2 persists session to `localStorage` by default — no custom storage needed.
- **Loading state:** On first render, session is `null` until `onAuthStateChange` fires. The `loading` flag prevents a flash of the login redirect before the session is confirmed.
- **AuthGuard:** Must handle the loading state before checking session, otherwise fresh page loads will always briefly redirect to `/login`.
- **Absolute imports:** Use `@/` alias for all imports (configured in `tsconfig.app.json` and `vite.config.ts`).
- **Spec ref:** Auth flow — `spec.md` Section 3.5

```typescript
// useAuth.ts pattern
export function useAuth() {
  const [session, setSession] = useState<Session | null>(null)
  const [loading, setLoading] = useState(true)

  useEffect(() => {
    const { data: { subscription } } = supabase.auth.onAuthStateChange((_event, session) => {
      setSession(session)
      setLoading(false)
    })
    return () => subscription.unsubscribe()
  }, [])

  const signIn = (email: string, password: string) =>
    supabase.auth.signInWithPassword({ email, password })

  const signOut = () => supabase.auth.signOut()

  return { session, loading, signIn, signOut }
}
```
