## Description

Assemble the complete "Nova OperaГ§ГЈo" flow by composing the pieces built in MVP-4-1 and MVP-4-2:

- **`OperacaoForm.tsx`** вЂ” The main form component using `useFieldArray` for dynamic legs. Handles `valorPagoFixo` symmetric propagation (when set, copies the value to every leg's `grossReturn`). Renders a `LegRow` per leg with Add/Remove controls, shows `PnLPreview`, and exposes an `onSubmit` prop.
- **`NovaOperacaoPage.tsx`** вЂ” The page that mounts `OperacaoForm`, wires `createOperacao` as `onSubmit`, implements draft recovery on mount (loads saved draft from `useOperacaoDraftStore`), saves draft on every change via `watch`, and clears the draft on successful submission.

Browser verification: Tab/Enter through a 3-leg ExtraГ§ГЈo operation end-to-end. Draft persists across page reload and is recovered automatically on remount (NFR-5).

  - **Tipo field:** `Select` component with the 5 `OperacaoTipo` options: `Freebet`, `Extracao`, `SuperOdd`, `TentativaDuplo`, `Aposta`. Controlled via `Controller`.

  - **Legs array:** `useFieldArray({ control, name: 'legs' })`. Default first leg appended on mount.
  - Each leg renders as `<LegRow index={i} ... onEnterLastField={() => append(emptyLeg)} onRemove={() => remove(i)} casas={casas} />`.
  - **"+ Adicionar Casa" button** (secondary) calls `append(emptyLeg)` вЂ” fallback for users not using Enter.
  - `emptyLeg` default: `{ casaId: '', stake: 0, grossReturn: 0, isFreebet: false, isDoubleGreen: false }`
  - **PnLPreview** rendered below the legs: `<PnLPreview legs={watchedLegs} casas={casas} />` where `watchedLegs = watch('legs')`.
  - **Submit button** labeled "Salvar OperaГ§ГЈo". While submitting: disabled and shows loading text.
  - `casas` fetched from `useCasas()` inside the component.

- [ ] **AC-2:** `src/pages/NovaOperacaoPage.tsx` exports `NovaOperacaoPage`:
  - Imports `useOperacaoDraftStore`, `useOperacoes`, `OperacaoForm`
  - **Draft recovery on mount:** `const { draft, clearDraft } = useOperacaoDraftStore()` в†’ passes `draft ?? undefined` as `defaultValues` to `OperacaoForm`
  - **Save draft on change:** Inside `OperacaoForm` (or via a wrapper effect), the page subscribes to form changes and calls `setDraft(currentValues)` on each change. Use the `onChange` callback approach: pass an `onChange` prop to `OperacaoForm` that the form calls via `watch` subscription.
    - Alternative simpler pattern: move the draft save logic into `OperacaoForm` as an optional `onDraftChange` prop.
  - **onSubmit handler:**
    1. Calls `createOperacao(data)` from `useOperacoes`
    2. On success: `clearDraft()`, show `toast.success('OperaГ§ГЈo salva!')`, navigate to `/operacoes`
    3. On error: show `toast.error(error.message)`
  - Page title: "Nova OperaГ§ГЈo"

- [ ] **AC-3:** `valorPagoFixo` symmetric propagation:

  - Zod schema already handles the constraint: if any leg has `grossReturn !== valorPagoFixo`, `valorPagoFixo` should be `null` (enforced by schema, not the UI propagation).

## Technical Notes

- **`useFieldArray` append on Enter:** `append({ casaId: '', stake: 0, grossReturn: 0, isFreebet: false, isDoubleGreen: false })` вЂ” match `legSchema` defaults. `useFieldArray` is imported from `react-hook-form`.
- **`valorPagoFixo` watch + setValue pattern:**
  ```typescript
  const watchedValorPagoFixo = watch('valorPagoFixo')
  useEffect(() => {
    if (watchedValorPagoFixo && watchedValorPagoFixo > 0) {
      fields.forEach((_, i) => setValue(`legs.${i}.grossReturn`, watchedValorPagoFixo))
    }
  }, [watchedValorPagoFixo])
  ```
- **`watch('legs')` for PnLPreview:** `const watchedLegs = watch('legs')` subscribes to leg changes. TanStack updates will re-render `PnLPreview` automatically.
- **Draft save via `watch` subscription (in NovaOperacaoPage):**
  ```typescript
  useEffect(() => {
    const subscription = form.watch((values) => {
      setDraft(values as OperacaoFormData)
    })
    return () => subscription.unsubscribe()
  }, [form.watch])
  ```
  Alternatively, pass `onDraftChange` as a prop to `OperacaoForm` and call it from within the form's `watch` subscription.
- **Router integration:** `src/router.tsx` (from MVP-2-1) should have a `/nova-operacao` route. If the route already exists as a stub, update it. If not, add it alongside the existing protected routes.
- **`casas` inside `OperacaoForm`:** `const { casas } = useCasas()` вЂ” the hook is cheap to call (TanStack Query caches it). This avoids prop drilling from `NovaOperacaoPage`.
- **Sonner toast:** `import { toast } from 'sonner'` вЂ” `toast.success('OperaГ§ГЈo salva!')`, `toast.error(error.message)`.
- **Navigation:** `import { useNavigate } from 'react-router-dom'` вЂ” `navigate('/operacoes')` on success.
- **`OperacaoTipo` options for Select:** The `Select` component accepts `options: { value: string; label: string }[]`. Use: `[{ value: 'Freebet', label: 'Freebet' }, { value: 'Extracao', label: 'ExtraГ§ГЈo' }, { value: 'SuperOdd', label: 'Super Odd' }, { value: 'TentativaDuplo', label: 'Tentativa Duplo' }, { value: 'Aposta', label: 'Aposta' }]`.
- **Spec refs:** FR-2 (new operation), FR-3 (5 types), FR-6 (Tab/Enter + valorPagoFixo), NFR-1 (keyboard-first), NFR-5 (draft recovery).
