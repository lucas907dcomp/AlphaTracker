## Description

Core da feature. Quando o usuário cria ou edita uma operação de `Extracao` ou `Freebet`, e a casa principal da operação possui um parceiro vinculado, o sistema exibe:

1. **Picker de origem** — dropdown com as freebets disponíveis para vincular (operações `FreebetSePerder` com `status = 'GeradaFreebet'` ou `Freebet` com `status = 'Concluida'` ainda não usadas como origem de outra operação)
2. **Campo de custo de liberação** — preenchido automaticamente a partir da operação de origem, editável manualmente
3. **SplitBreakdown** — exibe em tempo real: lucro bruto, custo, lucro líquido, parte do parceiro, parte do usuário

Ao salvar, os campos `operacao_origem_id` e `custo_liberacao` são persistidos na operação. A operação de origem é marcada como "usada" implicitamente (nenhum outro picker a mostrará).

## Technical Notes

- "Casa com parceiro" no formulário: verificar se `casas.find(c => legs.some(l => l.casaId === c.id))?.parceiro_id` é não nulo
- O `FreebetOrigemPicker` só aparece quando `tipo` é `Extracao` ou `Freebet` E a condição acima é verdadeira — caso contrário, `operacao_origem_id` fica null
- `useFreebetsDisponiveis` deve excluir operações já usadas: `supabase.from('operacoes').select().not('id', 'in', (select operacao_origem_id from operacoes where operacao_origem_id is not null))` — ou fazer em dois steps no client
- `CentavosInput` (`src/components/ui/CentavosInput.tsx`) deve ser usado para o campo `custoLiberacao`
- O `SplitBreakdown` é read-only, recalculado a cada mudança no formulário via `watch` do react-hook-form
- Para o join de parceiro no `OperacaoCard`: as apostas já carregam `casa:casas(*)`, e `casas` agora tem `parceiro:parceiros(*)` — a query em `useOperacoes` precisará ser atualizada para incluir esse join aninhado: `apostas(*, casa:casas(*, parceiro:parceiros(*)))`
