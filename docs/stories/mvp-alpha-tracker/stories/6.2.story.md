## Description

Extend the operations management flow with Edit and Delete actions (FR-11). This story delivers:

- **`ConfirmDialog.tsx`** — Generic confirmation modal component, reusable across the app.
- **`EditOperacaoModal.tsx`** — Modal wrapping `OperacaoForm` with existing operation data, calls `updateOperacao` on submit.
- **`useOperacoes.ts`** (extend) — Add `updateOperacao` mutation (update operacao row + delete apostas + re-insert apostas).
- **`OperacaoCard.tsx`** (extend) — Add Edit and Delete action buttons.
- **`OperacoesPage.tsx`** (extend) — Manage delete confirm state and edit modal state.

After delete, the Dashboard cache is invalidated automatically (already wired in `deleteOperacao`). After edit (update), the same invalidation ensures the list and dashboard reflect updated data.

## Technical Notes

- **`updateOperacao` step 1 — status reset:** If `operacao.status === 'Concluida'`, the update must reset `status = 'Pendente'` and `pnl = null`. Otherwise, the operation would remain "Concluida" with stale PnL after apostas change. Pattern:
  ```typescript
  const statusReset = operacao.status === 'Concluida' ? { status: 'Pendente', pnl: null } : {}
  await supabase.from('operacoes').update({
    tipo: data.tipo,
    data: data.data,
    valor_pago_fixo: data.valorPagoFixo ?? null,
    notas: data.notas ?? null,
    ...statusReset,
  }).eq('id', operacaoId)
  ```
  Wait — `updateOperacao` receives `OperacaoFormData` but needs to know the current status to decide the reset. Pass the existing `Operacao` object to the mutation params instead:
  ```typescript
  type UpdateOperacaoParams = {
    operacao: Operacao        // full object — needed for status check
    data: OperacaoFormData
  }
  ```
  Update AC-2 accordingly: use `operacao.status === 'Concluida'` from the params.
  
- **Delete apostas before re-insert:** The apostas FK to operacoes is `ON DELETE CASCADE` per schema design, but explicit delete before re-insert is safer: `supabase.from('apostas').delete().eq('operacao_id', operacaoId)`. If cascade is active, deleting the operacao and recreating is also valid — but we keep the operacao ID so explicit delete + re-insert preserves history.
- **OperacaoList prop changes:** Adding `onDelete` and `onEdit` to `OperacaoListProps` is a breaking change to the props interface. OperacoesPage must supply these. No other caller exists yet.
- **Spec refs:** FR-11 (edição e exclusão de Apostas e Operações após criação)
