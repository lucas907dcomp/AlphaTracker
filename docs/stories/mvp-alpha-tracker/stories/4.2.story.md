## Description

Three tightly coupled pieces that together enable fast mobile data entry and live PnL feedback:

1. **`operacaoDraftStore.ts`** — Zustand persist store that saves the OperacaoForm draft to localStorage (`name: 'operacao-draft'`). Enables draft recovery across page reloads (NFR-5).
2. **`LegRow.tsx`** — A single aposta row with Tab/Enter keyboard navigation: Tab moves through `casaId → stake → grossReturn → isFreebet → isDoubleGreen`, Enter on the last field triggers `onEnterLastField()` to add a new leg without mouse interaction (NFR-1).
3. **`PnLPreview.tsx`** — Real-time PnL preview that calls `calcularCenariosPnL` on every render of the current legs. Updates in < 100ms (NFR-4) because it is purely client-side computation with no server round-trip. Shows a table of scenarios colored green (pnl > 0) or red (pnl ≤ 0).

## Technical Notes

- **Zustand persist v4.5.x:** `import { create } from 'zustand'` + `import { persist } from 'zustand/middleware'`. The `persist` middleware takes `(set) => ({...})` as the first argument and `{ name: 'operacao-draft' }` as the second. The type parameter on `create` is the full store type.
- **`LegFormData` import:** Import from `@/schemas/operacaoSchema` — the inferred type from `legSchema`.
- **`PnLPreview` — `casaNome` fallback:** `casas.find(c => c.id === leg.casaId)` may return `undefined` if the leg's casaId doesn't match a loaded casa. The mapped `Aposta.casa` will be `undefined`, and `calcularCenariosPnL` handles this with `aposta.casa?.nome ?? ''`. Display the `casaNome` from the result, or fall back to "Casa" if empty.
- **`PnLPreview` — no memoization needed:** Calculation is < 1ms for 3-5 legs. React's normal render cycle is sufficient. Adding `useMemo` would be premature optimization.
- **`LegRow` — `valueAsNumber` on number inputs:** Without `{ valueAsNumber: true }`, react-hook-form will register the input as a string and Zod's `z.number()` validation will fail. Always use `register(\`legs.${index}.stake\`, { valueAsNumber: true })`.
- **`Control` and `UseFormRegister` types:** Import from `react-hook-form`: `import type { Control, UseFormRegister } from 'react-hook-form'`. The generic type is `OperacaoFormData` from `@/schemas/operacaoSchema`.
- **Tab order HTML:** The natural DOM tab order follows the render order. Render fields in the exact Tab order specified: Select (casaId) → Input (stake) → Input (grossReturn) → checkbox (isFreebet) → checkbox (isDoubleGreen). Do NOT use `tabIndex` unless needed to correct an existing order issue.
- **Select component:** `src/components/ui/Select.tsx` was created in MVP-2-2. Use `Controller` with `render={({ field }) => <Select {...field} options={...} />}`.
- **Spec refs:** NFR-1 (Tab/Enter), NFR-4 (< 100ms update), NFR-5 (draft persist), FR-4 (isFreebet toggle), FR-8 (isDoubleGreen).

```typescript
// operacaoDraftStore.ts — reference implementation
import { create } from 'zustand'
import { persist } from 'zustand/middleware'
import type { OperacaoFormData } from '@/schemas/operacaoSchema'

interface OperacaoDraftStore {
  draft: OperacaoFormData | null
  setDraft: (draft: OperacaoFormData) => void
  clearDraft: () => void
}

export const useOperacaoDraftStore = create<OperacaoDraftStore>()(
  persist(
    (set) => ({
      draft: null,
      setDraft: (draft) => set({ draft }),
      clearDraft: () => set({ draft: null }),
    }),
    { name: 'operacao-draft' }
  )
)
```
