## Technical Notes

- **vercel.json rewrites pattern:** The `"source": "/(.*)"` pattern with `"destination": "/index.html"` is the standard SPA rewrite for Vite apps on Vercel. This ensures `/operacoes`, `/dashboard`, etc. all return `index.html` (React Router handles routing client-side).
- **React.lazy with named exports:** `DashboardPage` uses `export default function DashboardPage()` — `lazy(() => import('@/pages/DashboardPage'))` works directly without `.then()` conversion.
- **Bundle size targets:** After lazy-loading DashboardPage (which brings recharts ~180KB), the main bundle should drop below 300KB gzip. If not, check for other large dependencies: date-fns (treeshakeable), sonner (small), react-hook-form (medium). Use `npx vite-bundle-analyzer` if needed.
- **Supabase CLI:** `npx supabase db push` requires Supabase CLI and a linked project. If Supabase CLI is not installed: `npm install -g supabase`. Alternative: apply migrations manually via Studio SQL editor (copy-paste each file).
- **Security reminder:** `.env` with `VITE_SUPABASE_ANON_KEY` must never be committed. The `_PROD` suffix naming convention (VITE_SUPABASE_URL_PROD) is the established project pattern for production credentials.
- **Spec refs:** NFR-2 (bundle < 300KB), NFR-3 (RLS prod isolation), CON-1 (SPA + Supabase Cloud + Vercel), CON-5 (Vite SPA deployed on Vercel)
