## Description

Create the Zod validation schemas used by `react-hook-form` throughout the application. `casaSchema` validates the CasaForm. `operacaoSchema` validates the NovaOperacaoPage form including dynamic leg rows. These schemas are the single source of truth for input validation тАФ they must cover all edge cases defined in the spec.

## Acceptance Criteria

- [x] **AC-1:** `src/schemas/casaSchema.ts` exports a Zod schema with:
  - `nome: z.string().min(1, 'Nome obrigat├│rio').trim()` (whitespace-only rejected)

- [x] **AC-2:** `src/schemas/operacaoSchema.ts` exports a Zod schema with:
  - `tipo: z.enum(['Freebet','Extracao','SuperOdd','TentativaDuplo','Aposta'])`

## Technical Notes

- Use `@hookform/resolvers/zod` in the consuming form components (not in this story).
- `legSchema` should be exported separately so `LegRow` component can type its props.
- The `casaId` field in `legSchema` references a UUID from the casas dropdown тАФ validation ensures no empty/invalid FK is submitted.
- `valorPagoFixo` is optional (null = asymmetric mode, value = symmetric mode). The distinction drives which PnL formula is used in `pnl.ts`.
